# secret-guard

`secret-guard` 是一个轻量级 Python 工具库，用于识别、扫描、脱敏和改写本地项目中暴露的机密信息与敏感基础设施信息。

这个项目主要服务于我自己的工具链。项目公开发布只是为了方便安装和复用，但在 `0.x` 版本期间 API 可能会随个人需求调整。


## 目标范围

`secret-guard` 主要关注这些类型的信息：

- API key
- access key
- token
- password
- credential
- private key
- authorization 值
- 账号类字段，例如 username、account、email
- 公网 IP
- 公网 IP 加非常见端口

它适合用于本地检查、开发辅助工具、仓库卫生检查。它不是专业密钥扫描工具、安全审计工具、合规审查工具或安全事件响应工具的替代品。

## 开发顺序

1. 识别

   识别是所有后续功能的基础。扫描、脱敏、改写都必须依赖统一的识别规则。

   - [x] 判断字段名是否像敏感字段，例如 `apiKey`、`access-token`、`clientSecret`
   - [x] 支持大小写、连接符、下划线、点号等命名差异
   - [x] 支持项目侧传入额外敏感字段集合
   - [x] 识别常见赋值语法，例如 `key=value`、`key: value`、带引号的配置值
   - [ ] 识别高置信密钥值，例如常见 API key、云厂商 key、GitHub token、私钥头
   - [ ] 区分密钥类、账号类、网络地址类等敏感类别
   - [ ] 识别公网 IP
   - [ ] 识别公网 IP 加非常见端口
   - [ ] 区分常见公共 IP 与需要关注的公网 IP
   - [ ] 过滤明显占位值，例如 `your-api-key`、`example`、`test-key`
   - [x] 避免把 `max_tokens`、`tokens_count` 这类计数字段误判为 token
   - [ ] 支持调用方配置额外规则、忽略规则和常见端口列表
   - [ ] 为识别规则补完整测试用例

2. 扫描

   扫描负责把识别能力应用到具体输入源上。扫描结果不能暴露原始敏感值。

   - [ ] 扫描纯文本
   - [ ] 扫描单个文件
   - [ ] 扫描目录
   - [ ] 区分配置文件、代码文件、数据库文件和普通文本文件
   - [ ] 跳过常见依赖目录和缓存目录
   - [ ] 跳过或限制大型文件、二进制文件
   - [ ] 对二进制或大文件只做高置信模式扫描，避免完整解析
   - [ ] 扫描 SQLite key/value 表
   - [ ] 扫描 Git 历史
   - [ ] 支持只扫描工作区、不扫描 Git 历史
   - [ ] 支持传入扫描根目录和排除路径
   - [ ] 输出结构化发现结果，包括路径、行号、类别、不可逆指纹
   - [ ] 合并重复发现，避免同一位置重复报告

3. 脱敏

   脱敏负责把扫描结果或任意值安全展示出来。它依赖识别结果，不应该绕过识别规则自己判断一套逻辑。

   - [ ] 根据字段名隐藏敏感值
   - [ ] 对文本里的 `token=...`、`password:...` 等片段做内联脱敏
   - [ ] 对 URL 中的用户名密码片段做脱敏
   - [ ] 支持最大输出长度，避免长提示词或长配置刷屏
   - [ ] 支持把多行值压缩为单行预览，避免提示词或证书内容刷屏
   - [ ] 支持自定义脱敏占位符，例如 `[secret hidden]`
   - [ ] 支持统一的输出对象格式
   - [ ] 支持文本、JSON 等输出格式
   - [ ] 支持稳定但不可逆的敏感值指纹
   - [ ] 确保扫描报告、日志和异常信息都不输出原始敏感值

4. 改写

   改写是最高风险功能，必须建立在扫描结果和脱敏稳定之后。

   - [ ] 根据扫描结果生成改写计划
   - [ ] 默认 dry-run，只展示将修改什么
   - [ ] 显式确认后才允许原地修改
   - [ ] 支持写入备份文件
   - [ ] 支持把敏感值替换为占位符
   - [ ] 支持移除敏感值
   - [ ] 支持只改写明确命中的文本范围，避免整文件格式被重写
   - [ ] 支持改写前后 diff 预览
   - [ ] 防止误改二进制文件、依赖目录、Git 内部目录
   - [ ] 改写失败时保留原文件，避免部分写入造成文件损坏

## 安装

从本地仓库安装：

```bash
pip install -e .
```

从 GitHub 安装：

```bash
pip install git+https://github.com/your-name/secret-guard.git
```

安装固定 tag：

```bash
pip install git+https://github.com/your-name/secret-guard.git@v0.1.0
```

## 命名约定

- 仓库名：`secret-guard`
- Python 包名：`secret-guard`
- Python 导入名：`secret_guard`
- CLI 命令：`secret-guard`

## 安全边界

`secret-guard` 使用启发式规则。这意味着：

- 可能存在误报
- 可能存在漏报
- 它应该作为额外安全层，而不是唯一保护手段
- 报告和日志里不应该直接输出敏感原值
- 改写功能必须默认 dry-run，不能默认进行破坏性修改
